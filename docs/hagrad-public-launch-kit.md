# HAGRad Public Launch Kit

## Goal
Publish HAGRad as a scientific, non-commercial research platform with:

- a public-facing website
- downloadable local apps
- clear research-use-only language
- citation guidance for papers and presentations

This approach is stronger than a browser-only upload tool because image processing stays local on the user's machine.

## Recommended public model

### Website
Use the website for:

- project overview
- screenshots and short demo clips
- workflow descriptions
- publications and citations
- downloads
- disclaimer and license text
- contact information

### Downloadable local tools
Distribute the actual tools as local downloads, for example:

- `HAGRad Viewer`
- `HAGRad Viewer - EAT Workflow`
- `HAGRad Viewer - Coronary Workflow`
- `HAGRad Viewer - QCA Workflow`
- `HAGRad Stent Viewer`
- `HAGRad CalcScorer`

### Research positioning
Position HAGRad as:

- a scientific imaging workstation
- a research and education tool
- not a clinical decision system
- not a regulated medical device offering

## Recommended website structure

### 1. Home
Short description of HAGRad, the imaging modalities, and the main workflows.

Suggested headline:

`HAGRad is a research imaging platform for advanced cardiovascular image analysis, presentation, and quantitative workflow development.`

Suggested subheadline:

`Built for scientific use in cardiac CT and angiographic imaging, HAGRad combines viewer-based review, multi-planar reformations, quantitative analysis, export workflows, and dedicated research modules in a local, non-commercial environment.`

### 2. Workflows
One page or section per workflow:

- HAGRad Viewer
- EAT Workflow
- Coronary Workflow
- QCA Workflow
- Stent Viewer
- CalcScorer

For each workflow show:

- what goes in
- what can be measured
- what gets exported
- current maturity and limitations

### 3. Download
Include:

- current release number
- supported operating systems
- install guide
- release notes
- checksum if you want a more professional distribution

### 4. Documentation
Include:

- quick start
- keyboard shortcuts
- export format descriptions
- troubleshooting
- example cases

### 5. Publications
Include:

- associated papers
- abstracts
- posters
- conference presentations
- citation text

### 6. Disclaimer / License
This page should be easy to find and linked in the footer.

### 7. Contact
For collaboration, bug reports, and academic inquiries.

## Domain strategy
You said you would be willing to pay for a good domain. I agree that this is worthwhile because a strong domain makes the project look much more credible and citable.

Good styles:

- short and memorable
- clearly connected to HAGRad
- easy to say and spell
- not tied to one narrow workflow

Suggested domain patterns to consider:

- `hagrad.org`
- `hagradviewer.org`
- `hagradlab.org`
- `hagradresearch.org`
- `hagrad-imaging.org`
- `hagrad.tools`

I have not checked live availability here. If needed, that can be done separately before purchase.

My preference would be:

1. `hagrad.org`
2. `hagradlab.org`
3. `hagradresearch.org`

Reason:

- `.org` fits scientific and non-commercial positioning well
- the names are broad enough to cover multiple workflows over time

## Licensing recommendation

If you want the tool to be `non-commercial`, the cleanest practical option is usually:

- a custom `research-use-only, non-commercial, source-available license`

That is different from standard open-source licensing.

Recommended positioning:

- source available for scientific inspection and academic collaboration
- non-commercial use only
- no clinical use
- no warranty

## Suggested plain-language license/distribution model

### Recommended policy

- free for academic and non-commercial scientific use
- no use for direct clinical decision-making
- no resale, paid redistribution, or commercialization without written permission
- no warranty or liability

### Suggested website wording

`HAGRad is provided for research, education, and software evaluation purposes only. It is not intended for clinical diagnosis, treatment planning, or patient management. Use of HAGRad is limited to non-commercial scientific and educational settings unless separate written permission is granted by the authors.`

### Suggested download checkbox wording

`By downloading HAGRad, you agree that this software is provided for non-commercial research and educational use only, is not a medical device, and must not be used as the sole basis for clinical decision-making.`

## Research-use disclaimer

Suggested long-form disclaimer:

`HAGRad is a scientific software platform intended for research, technical development, education, and retrospective image analysis. It has not been validated as a clinical diagnostic product and must not be used as the sole basis for diagnosis, treatment selection, procedural planning, or other patient-care decisions. The software is provided without warranty of performance, safety, or fitness for a particular purpose. Users are responsible for complying with all institutional, ethical, privacy, and regulatory requirements applicable to their use of medical imaging data.`

## Citation strategy

You should prepare three ways to cite HAGRad:

### 1. Software citation
For example:

`HAGRad Viewer, version X.Y, Hagar MT et al., year.`

### 2. DOI citation
Archive releases through Zenodo or a similar service.

### 3. Manuscript citation
Reference the main paper describing the software and its workflows.

## Launch checklist

Before public release:

- choose domain
- choose license model
- write disclaimer
- prepare screenshots
- write quick start guide
- produce a first release package
- assign version number
- archive release with DOI
- prepare manuscript or technical note

## Practical recommendation

If you want a strong first public launch, the best order is:

1. pick domain and branding
2. finalize license/disclaimer
3. create a clean public website
4. package one stable public release
5. publish the accompanying paper or preprint

That sequence will make HAGRad look like a serious scientific software project rather than just a personal prototype.
